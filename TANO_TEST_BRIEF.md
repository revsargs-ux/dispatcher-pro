## 📋 Полный сценарий тестирования (E2E)

### Шаг 1 — Регистрация рабочего
- Зайти на worker.html
- Нажать "Зарегистрироваться"
- Заполнить ФИО, телефон (уникальный), пароль
- Убедиться что вошёл в кабинет после регистрации
- ✅ Ожидаемо: профиль отображается

### Шаг 2 — Вход диспетчера
- Зайти на https://диспетчер-про.рф
- Войти: +70000000003 / Test1234
- ✅ Ожидаемо: открылась панель диспетчера

### Шаг 3 — Создание клиента (если нет)
- Вкладка "Заказчики" → "Добавить"
- Заполнить название, телефон
- ✅ Ожидаемо: клиент появился в списке

### Шаг 4 — Создание смены
- Кнопка "Новая смена"
- Выбрать клиента, дату, время, услугу, количество рабочих
- Нажать "Далее" → выбрать рабочего из списка → "Пригласить"
- ✅ Ожидаемо: смена создана, рабочему отправлено приглашение

### Шаг 5 — Рабочий принимает смену
- Войти как рабочий (+70000000001 / Test1234)
- Увидеть смену в кабинете
- Нажать "Принять"
- ✅ Ожидаемо: статус сменился на "Подтверждён"

### Шаг 6 — Рабочий начинает/заканчивает смену
- Нажать "▶ Начать работу"
- Подождать (или сразу) нажать "⏹ Завершить"
- ✅ Ожидаемо: время зафиксировано

### Шаг 7 — Диспетчер вводит часы
- Вернуться в панель диспетчера
- Открыть смену → нажать ⏱ рядом с рабочим
- Ввести время начала/конца → Сохранить
- ✅ Ожидаемо: часы сохранены, клиенту ушло уведомление (Telegram/МАКС)

### Шаг 8 — Клиент подтверждает часы (НОВЫЙ ФУНКЦИОНАЛ)
- Войти как клиент (+70000000002 / Test1234)
- Увидеть блок "👷 Рабочий — Xч" с кнопками
- **Вариант А:** нажать ✅ Подтвердить → проверить что у диспетчера разблокировалась 💰
- **Вариант Б:** нажать ✏️ Оспорить → ввести другое кол-во часов → проверить что диспетчер видит спор
- ✅ Ожидаемо: статус обновился, уведомления отправлены

### Шаг 9 — Автоподтверждение (24ч)
- Тестировать через API: создать запись с hours_submitted_at = 25 часов назад
- Дождаться следующего цикла autoConfirmHours (каждые 30 мин)
- ✅ Ожидаемо: статус стал auto_confirmed, клиенту пришло уведомление

### Шаг 10 — Диспетчер закрывает смену и оплачивает

- Завершить смену (✖ Закрыть)

## 🐛 Что проверить отдельно

- [ ] Регистрация с уже существующим телефоном → должна быть ошибка
- [ ] Вход с неверным паролем → сообщение об ошибке
- [ ] Рабочий не видит данные других рабочих
- [ ] Клиент не видит данные других клиентов
- [ ] Диспетчер видит только свои смены
- [ ] Кнопка 💰 недоступна пока клиент не подтвердил часы
- [ ] Через 24ч без ответа клиента → автоподтверждение

## 📊 API для проверок

```bash
# Проверить смены
curl "https://диспетчер-про.рф/api/shifts?select=*&limit=5" -H "Authorization: Bearer <TOKEN>"

# Проверить назначения
curl "https://диспетчер-про.рф/api/shift_assignments?select=*&limit=5" -H "Authorization: Bearer <TOKEN>"

# Тест подтверждения часов
curl -X POST "https://диспетчер-про.рф/api/client-hours-confirm" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <TOKEN>" \
  -d '{"assignment_id":"<ID>","action":"confirm"}'
```
